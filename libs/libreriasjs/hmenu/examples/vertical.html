The menu on the left is created in vertical state by default. As you can see in this page source, it is defined in the exact same way as a horizontal menu, the only difference being at creation time: you need to pass "vertical: true" in the configuration parameter.

As it is more intuitive this way, a vertical menu will also imply “electric” behavior (unless you explicitely say “electric: false” at configuration).

Note, in the source of this page, how easy can we style the base vertical menu. We just need to assign a certain class name to the containing element (the element is here a <td> and the class name is “leftside”), and based on that we can write CSS rules (see in the <head> of the document) that apply only to what is inside, that is, our vertical menu.

More interesting things to consider:

  • The menu width and height are determined automatically, based on the content width—so you need not to mess with absolue values that would render your pages almost useless if the end-user's machine doesn't have the exact same fonts, etc.
  • For a proper layout, you should—in the case of a vertical menu—embed the toplevel <UL> element in either a table column of a DIV with a floating style. We used the table approach here, because it seems to be supported better by Internet Explorer.

Just to demonstrate that it's the exact same HTML code (except for the ID which has to be unique throughout the document), following we include the same menu in it's horizontal form. See the page source to see how it's done.
